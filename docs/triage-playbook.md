# Triage Playbook

Use this lightweight loop for public feedback.

## Weekly Loop

1. Review new issues and discussions.
2. Add `needs-triage`, `bug`, `idea`, `q-a`, `beta`, `needs-info`, or `private-support`.
3. Move questions from Issues to Discussions when there is no reproducible bug.
4. Move private, billing, privacy, or security reports to support@bonjour.dev.
5. Convert accepted ideas or confirmed bugs into implementation work in the private product repository.
6. Close the loop by replying when a fix ships or a decision is made.

## Status Labels

- `needs-triage`: New item, not reviewed yet.
- `needs-info`: Waiting for the reporter.
- `planned`: Accepted for a future release.
- `shipped`: Released in App Store or TestFlight.
- `wontfix`: Not aligned with current Expose scope.

## Privacy Rules

Ask users to remove or move the conversation to email when a report includes:

- Access tokens or URLs with `token=`.
- Private local network details.
- Personal files, photos, videos, or screenshots.
- Purchase receipts or account-specific payment details.
- Security vulnerabilities.
