# Expose Community & Feedback

This is the official public feedback hub for Expose, the local-first HTTP/API server for iPhone and iPad.

Use this repository for:

- Public product questions and workflow help.
- Feature ideas and roadmap input.
- Reproducible bug reports.
- Beta testing feedback.
- Sharing how you use Expose in local development, QA, automation, classrooms, labs, or media workflows.

Please do not post private files, access tokens, local network URLs that include tokens, screenshots with sensitive data, crash logs containing personal data, or security vulnerabilities.

## Where To Go

- Ask a question: [Q&A](https://github.com/bonjour-studio/expose-feedback/discussions/categories/q-a)
- Suggest an idea: [Ideas](https://github.com/bonjour-studio/expose-feedback/discussions/categories/ideas)
- Share a workflow: [Show and tell](https://github.com/bonjour-studio/expose-feedback/discussions/categories/show-and-tell)
- Report a bug: [New issue](https://github.com/bonjour-studio/expose-feedback/issues/new/choose)
- Private support: [support@bonjour.dev](mailto:support@bonjour.dev)
- Privacy Policy: [expose.bonjour.dev/privacy](https://expose.bonjour.dev/privacy)
- Terms of Use: [expose.bonjour.dev/terms](https://expose.bonjour.dev/terms)

## Good Feedback Includes

- What you were trying to do.
- What happened instead.
- Expose version and build number.
- Device model and iOS/iPadOS version.
- Whether the issue involves routes, sources, files, photos, videos, logs, networking, purchases, or localization.
- A minimal route/source setup that reproduces the issue, if possible.

## Triage Labels

- `needs-triage`: New report that needs maintainer review.
- `needs-info`: Waiting for reproduction details or clarification.
- `bug`: Confirmed or likely product bug.
- `idea`: Product idea or feature request.
- `q-a`: Public support question.
- `beta`: Feedback about TestFlight or pre-release builds.
- `planned`: Accepted for a future release.
- `shipped`: Released in the App Store or TestFlight.
- `wontfix`: Not aligned with Expose product scope.
- `private-support`: Should move to email because it involves private data, billing, or account-specific details.
- `security`: Security-sensitive; public discussion should stop and move to email.
